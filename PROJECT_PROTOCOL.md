# 项目协议 | Project Protocol

## 1. 目标

本协议用于约束本仓库在工业通信场景下的开发、发布与协作行为，确保代码可维护、配置可迁移、接口可兼容。

## 2. 开发边界

- 核心库目录：`MODBUS-LIB`
- 示例工程目录：`Examples`
- 测试脚本目录：`Script`
- 文档目录：`docs`

任何改造必须优先保证：

1. 现有示例工程可继续导入并编译
2. 已公开的核心结构体字段不做破坏性删除
3. 关键行为变更需在 README 和变更日志同步说明

## 3. 分支与提交规范

- 功能分支命名：`feature/<short-name>`
- 修复分支命名：`fix/<short-name>`
- 文档分支命名：`docs/<short-name>`
- 提交信息建议采用：`type(scope): summary`

## 4. 配置与安全规范

- 禁止提交本机绝对路径
- 禁止提交局域网内网地址作为默认生产配置
- 禁止提交真实串口资产清单或设备凭据
- 所有本地化参数统一进入模板文件或环境变量

## 5. 兼容性策略

- 默认保持 RTU 主从逻辑向后兼容
- 新特性优先通过宏开关引入
- 不破坏已有 `ModbusConfig.h` 的基础宏语义

## 6. 文档策略

- 对外文档以中文为主，关键术语保留英文
- 所有新增模块必须附最小可运行示例
- 关键配置变更需补充“迁移说明”

## 7. 发布策略

- 发布前至少完成一次串口回环或双设备联调
- 发布前确认脚本与示例不含个人路径
- 发布版本建议携带 `Release Notes`

## 8. 许可证

本仓库沿用根目录 `LICENSE` 约定。
